Is Your Database Slowing Down Your Drupal Site?

Often overlooked in the "Varnish vs. Redis" debate is a simpler, highly effective alternative for sites that don't have complex server stacks: FileCache.

If you are running a Drupal site on a standard VPS or shared hosting, you probably rely on the default database cache. But if you aren't using an in-memory store like Redis, swapping your database cache for FileCache can be one of the easiest "quick wins" for performance.

Here is a breakdown of how FileCache compares to the standard Drupal cache and why it might be the best fit for your site.

1. The Standard: Drupal Database Cache

By default, Drupal stores its "cache bins" (where it keeps things like rendered blocks, views, and page fragments) inside your SQL database (MySQL or MariaDB).

How it works: Every time a visitor loads a page, Drupal asks the database for the cached version of various page parts.

The Problem: On busy sites, the database becomes a bottleneck. It has to handle both "real" data (your nodes and users) and "temporary" data (the cache). This creates high "IO" (input/output) strain.

Best for: Very small sites with low traffic or developers who want zero configuration.

2. The Alternative: FileCache Module

The FileCache module moves those cache bins out of your database and onto your server's hard drive (or SSD) as simple files.

How it works: Instead of making a SQL query to find a cached block, Drupal just reads a file from a folder.

The Advantage: Modern SSDs are incredibly fast at "reading" small files. By moving the cache to the file system, you liberate your database to focus solely on critical tasks like saving content or processing orders.

Best for: Sites on SSD-based hosting that want a speed boost without the headache of managing Redis or Varnish.

Why focus on FileCache if you use a CDN?

Many site owners think, "I have Cloudflare (CDN), so my cache doesn't matter." This is a mistake.

A CDN only caches the entire page for anonymous users. But as soon as a user logs in, or if the CDN cache "misses," the request hits your server. If your server then has to struggle through thousands of database queries just to build the page, the user experience suffers.

FileCache ensures that when your server does have to work, it does so as efficiently as possible, without the complexity of a Varnish/Redis setup.

Is FileCache right for your website?

✅ Use FileCache if:

You are on Shared Hosting or a basic VPS: You likely don't have permission to install Redis or Varnish, but you do have permission to write files to a folder.

Simplicity is Priority: You aren't a "DevOps" person. You want a "set it and forget it" solution that doesn't require managing a separate server process.

Your Database is Slow: If you notice your site drags whenever the database is under load, moving the cache to the file system provides instant breathing room.

Small to Medium Sites: For sites with 1,000–50,000 visitors a month, FileCache is often "good enough" to rival Redis.

❌ Stick to Regular (or move to Redis) if:

You have a Multi-Server Setup: If you have multiple web servers behind a load balancer, FileCache won't work easily because the files reside on one specific server. You need a shared memory store like Redis.

HDD (Not SSD) Storage: If you are on very old "spinning disk" hardware, reading files might actually be slower than the database.

Extreme Traffic: If you are the next New York Times, you need the sub-millisecond speeds of RAM (Redis).

How to Switch (The Non-DevOps Guide)

You don't need a terminal to get started.

Install the module: Download and enable the FileCache module.

Edit your settings.php: You will need to add a few lines of code to your settings.php file to tell Drupal to stop using the database and start using the folder.

$settings['cache']['default'] = 'cache.backend.file_system';
$settings['filecache_paths']['default'] = '../directory_outside_webroot/filecache';

Clear Caches: Go to your performance page, hit "Clear all caches," and you're done!

Final Verdict

If you’re running a standard Drupal site and want better performance without becoming a server administrator, FileCache is a hidden gem. It bridges the gap between the "too slow" database cache and the "too complex" Redis setup, giving you a faster site with almost zero maintenance.
